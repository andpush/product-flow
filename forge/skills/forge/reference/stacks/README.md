# Stack seeds

Starting points for a project's `rules*.md`, **loaded only during `forge ground` (greenfield)**.
They are examples of defensible defaults, not law — `ground` reads the matching seed, adapts it
with the user, and writes the trimmed result into the project. Edit freely; delete what doesn't
apply.

Keep this set small and curated. If maintaining a seed becomes a burden, drop it — these are a
convenience, not load-bearing. Add a new seed only when you actually start projects on that stack.

- [kotlin.md](kotlin.md) — JVM / Kotlin services
- [typescript.md](typescript.md) — TypeScript (Node, web, Svelte/React)
